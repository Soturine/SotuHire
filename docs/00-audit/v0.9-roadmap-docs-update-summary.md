# Atualização de documentação pós-v0.9.0

## Objetivo

Este documento registra a atualização de documentação feita após revisar o estado v0.9.0 do SotuHire e consolidar a próxima direção do projeto.

A atualização teve quatro focos:

1. corrigir a direção do roadmap pós-v0.9.0;
2. transformar o SotuHire em produto multiárea;
3. planejar prompts estruturados e completos;
4. evoluir a análise GitHub/portfólio para um pipeline mais profundo.

## Escopo respeitado

Esta atualização **não altera** a documentação de coleta autenticada.

Também **não cria** o documento `docs/ethics/allowed-vs-not-allowed.md`.

O escopo ficou restrito a:

- estratégia de produto;
- prompts;
- IA estruturada;
- matching;
- multiárea;
- GitHub Analyzer;
- roadmap de desenvolvimento.

## Decisões registradas

### SotuHire multiárea

O produto não deve ser vendido ou implementado apenas como ferramenta para Dev/TI.

Ele deve funcionar com vagas e currículos de múltiplas áreas, desde que o sistema classifique requisitos por tipo e não dependa apenas de keywords.

### IA com contrato

A IA deve receber entrada rica e devolver JSON validado.

O código deve validar, calcular score, aplicar travas e salvar histórico.

### Extensão como ponte

A extensão deve capturar contexto e chamar o SotuHire local/site.

A análise pesada deve ficar no backend, onde é possível reaproveitar memória, tracker, currículo, vaga, GitHub e prompts versionados.

### GitHub Analyzer 2.0

A análise de GitHub deve evoluir de sinais visíveis e heurística simples para:

- GitHub API;
- árvore completa;
- sampler;
- grafo de dependências;
- contexto de arquivos;
- prompt JSON;
- scores por dimensão;
- evidências por arquivo;
- comparação com vaga e currículo.

### Match Engine 2.0

O match deve evoluir para análise por requisitos, domínio, evidência e confiança.

A IA pode classificar e explicar, mas o score final deve ser calculado pelo código.
