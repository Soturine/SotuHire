# Auditoria de prontidão — SotuHire v0.4

## Escopo

Auditoria da entrega conjunta:

```text
v0.2 — UX/UI e automação básica
v0.3 — IA estruturada, Resume Tailor e exports
v0.4 — tracker local, histórico e dashboard
```

## Baseline

Antes desta evolução:

- arquivos rastreados: 152;
- Markdown rastreados: 74;
- testes: 39 passed;
- Ruff: passou;
- MkDocs strict: passou;
- Streamlit health check: ok.

## Auditoria inicial

Gaps encontrados:

- interface parecida com formulário de teste;
- preenchimento manual excessivo;
- sem parser de vaga;
- currículo limitado a TXT;
- sem provider estruturado/fallback;
- Resume Tailor sem bullets/evidências explícitas;
- sem exports;
- sem histórico;
- sem tracker funcional;
- sem dashboard.

## Gaps corrigidos

- interface guiada com seis abas;
- modo rápido e avançado;
- parsers TXT/PDF/DOCX e descrição de vaga;
- revisão assistida;
- mock provider e Gemini opcional;
- exports JSON/Markdown;
- tracker JSON local e privacidade explícita;
- histórico, status e dashboard;
- testes obrigatórios adicionados.

## Segurança e limites

- nenhum auto-apply;
- nenhum envio automático para recrutador;
- nenhum scraping agressivo;
- nenhuma coleta privada/logada;
- PyTorch fora das dependências obrigatórias;
- Concurso Mode continua futuro experimental;
- texto bruto do currículo não é salvo no histórico.

## Docs auditados

README, CHANGELOG, MkDocs, docs index, roadmap, implementação v0.1, regras do Resume Tailor e Opportunity Fit, provider strategy e Gemini Structured Output foram expandidos.
