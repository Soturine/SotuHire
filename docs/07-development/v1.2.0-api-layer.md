# v1.2.0 - API Layer / FastAPI Foundation

## Objetivo

Implementar uma FastAPI local, frontend-ready, sem quebrar o Streamlit nem a Local Companion API.

A v1.2.0 cria a ponte real entre o core Python e um frontend moderno futuro. A camada HTTP fica em
`apps/api`, mas as regras continuam em `modules/`.

## Como rodar

```bash
python scripts/run_api.py
```

Defaults:

```text
http://127.0.0.1:8787
```

OpenAPI:

```text
http://127.0.0.1:8787/openapi.json
http://127.0.0.1:8787/docs
```

Variaveis opcionais:

```env
SOTUHIRE_API_HOST=127.0.0.1
SOTUHIRE_API_PORT=8787
SOTUHIRE_API_ALLOWED_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

## Decisoes

- FastAPI e local por default.
- CORS restrito, configuravel por env.
- OpenAPI e fonte de verdade para clients HTTP.
- `ApiEnvelope` padroniza `ok`, `data`, `warnings` e `request_id`.
- `raw_text` nao volta nas respostas de extracao, exceto quando `include_raw_text=true`.
- GitHub Analyzer pode usar `fallback_payload` para cenarios sem API disponivel.
- Tracker usa o mesmo `JobTracker` e o mesmo store local.
- Application Intelligence fica em funcoes puras no backend, nao no frontend.

- client HTTP tipado;

- telas de resume extraction, job extraction, match, ATS, Tailor, GitHub Analyzer e tracker;
- consumo dos endpoints reais sem recalcular regra critica no frontend.
