# v1.7.1 - Intake Polish, Encoding Fixes & Source Discovery Prep

A v1.7.1 é uma versão de polimento do fluxo de entrada de oportunidades criado na v1.7.0. O foco é
melhorar confiança operacional: encoding correto, upload real pelo navegador, separação clara entre
Demo/API Real, mescla visual de duplicatas, exportação da caixa de entrada e preparação segura para
fontes públicas/oficiais.

## Objetivo da versão

- corrigir mojibake em textos públicos e adicionar regressão automática;
- impedir mistura silenciosa de dados fictícios quando o app está em API Real;
- permitir upload de CSV/JSON com preview e confirmação;
- preservar histórico em mesclas visuais de duplicatas;
- exportar oportunidades locais em CSV/JSON;
- preparar um Diretório de Fontes sem crawler amplo, bypass ou automação de conta.

## Endpoints adicionados

```txt
POST /api/v1/sources/captures/{capture_id}/merge
GET  /api/v1/sources/directory
POST /api/v1/sources/export
```

## Segurança e limites

- não salva API key no frontend;
- não retorna API key ao frontend;
- não captura cookie, token ou sessão;
- não altera authenticated browser, Chromium/CDP ou scraper autenticado existente;
- não faz bypass de CAPTCHA;
- não faz auto-apply;
- se uma página exigir login ou bloquear leitura pública simples, o usuário deve revisar manualmente
  e colar o texto da vaga.

## IA opcional

O prompt `source_import_enrichment_v1` pode sugerir tags, domínio, senioridade provável, prioridade,
resumo curto e alertas. Ele não pode inventar requisitos, empresa, salário, licença, certificação,
experiência ou status de candidatura.

Se `use_ai=false`, se o provider não estiver configurado ou se a chamada falhar, a importação usa
parser local e registra warning de fallback quando aplicável.
