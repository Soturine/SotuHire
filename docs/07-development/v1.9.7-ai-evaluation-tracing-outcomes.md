# Implementação v1.9.7 — avaliação, tracing e outcomes

A v1.9.7 adiciona registry de 16 tarefas, AiRunStore 2.0, datasets golden multiárea, métricas determinísticas, benchmark reprodutível, feedback humano, Outcome Learning e painel `/ai-quality`.

## Persistência

A migration 4 adiciona metadados seguros a `ai_runs` e cria `ai_feedback`, `ai_benchmarks`, `ai_benchmark_results`, `outcome_events` e `outcome_metrics`. Migrações continuam versionadas, idempotentes e precedidas por backup no fluxo operacional. O desenvolvimento da release não aplica a migration automaticamente à base pessoal existente.

O rollback operacional usa o ZIP criado antes do apply: primeiro execute o restore em dry-run e, após revisar checksums/schema, confirme o apply. Não existe down-migration destrutiva automática; essa escolha preserva dados e evita apagar traces/feedback/outcomes criados após a migration.

## API

Os endpoints de qualidade são paginados e nunca retornam entrada/saída sensível completa. Outcomes aceitam apenas eventos manuais; não existe auto-apply, decisão final de elegibilidade ou atualização automática do Perfil.

## Compatibilidade

Demo, API real e fallback local permanecem disponíveis. JSON/JSONL legado não foi removido. A extensão passou a 0.9.3 pelo novo feedback seguro. Os endpoints e módulos de authenticated-browser não foram alterados.
