# v1.9.1 - Release Integrity, README Overhaul, Context Unification & Extension Profile Bridge

## Objetivo

A v1.9.1 e uma versao de integridade, documentacao profissional, screenshots atuais, unificacao de
contexto e ponte revisavel entre extensao e Perfil Profissional Universal. Ela nao cria um novo
fluxo grande de candidatura.

## Entregas

- tag `v1.9.0` revisada para apontar ao commit final com lockfile validado;
- `npm ci`, build, lint e typecheck do frontend confirmados antes da retag;
- `README.md` reestruturado como pagina de produto/repo, nao como changelog;
- screenshots e GIF do frontend web moderno atualizados sem nome preso a versao;
- `modules/context` criado como Career Context Engine;
- Match, ATS, Tailor, Radar, Wishlist, Scheduler, Fontes, Tracker e GitHub passam a consultar contexto unificado;
- Extensao Local gera candidatos revisaveis para o Perfil a partir de capturas e projetos;
- RAG local lexical integrado ao contexto;
- testes adicionados para engine, APIs de contexto e README;
- roadmap, mapa de integracao, indice documental e release notes atualizados.

## Fora de escopo

- candidatura automatica;
- auto-apply;
- crawler amplo;
- bypass de CAPTCHA;
- captura de cookies, tokens, sessao, headers ou storage;
- mudanca no fluxo `/api/v1/sources/authenticated-browser/collect`;
- remover Streamlit legado/dev.

## Decisoes tecnicas

O engine e deliberadamente pequeno:

- usa `ProfileContextOrchestrator`;
- usa `MemoryRetriever`;
- deduplica evidencias localmente;
- retorna modelos Pydantic serializaveis;
- deixa formatadores decidirem o que pode virar prompt;
- nao cria banco vetorial nem dependencia externa.

## Integracoes iniciais

- Wishlist recebe `CareerContextPurpose.WISHLIST`;
- Radar manual e Scheduler usam `CareerContextPurpose.RADAR`;
- Scheduler usa contexto tambem para mensagens de notificacao;
- Match converte contexto em `CareerEvidence` para o Match Engine;
- ATS retorna `context_evidence_keywords`;
- Tailor usa contexto no ajuste local e respeita provider externo;
- Tracker retorna `context_summary`, `fit_reason`, `next_action_hint` e `recurring_gaps`;
- Fontes persistem metadata `career_context` em itens de inbox;
- GitHub retorna `profile_evidence_candidates` revisaveis.
- Extensao usa `CareerContextPurpose.EXTENSION`, expõe `/api/v1/extension/context` e salva no Perfil
  apenas candidatos selecionados pela pessoa usuaria.

## Pendencias futuras

- matching adaptativo por dominio pode ser aprofundado;
- upload direto de arquivos para Perfil pode evoluir;
- GitHub/Portfolio pode ganhar revisao ainda mais rica de autoria, escopo e evidencias;
- notificacoes nativas do sistema operacional permanecem futuras.
