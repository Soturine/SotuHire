# v1.7.0 - Public Sources, Importers & Capture History

A v1.7.0 transforma **Fontes e Captura** em um fluxo pratico de entrada de oportunidades. O foco foi
importar vagas por texto, link publico simples, CSV, JSON e capturas da extensao/local companion,
manter historico persistente, detectar duplicatas e conectar os itens ao fluxo Vaga -> Analise ->
Candidaturas.

## Objetivo

- reduzir trabalho manual repetitivo ao organizar vagas;
- preservar origem, texto original e link da oportunidade;
- revisar antes de analisar ou salvar no tracker;
- manter tudo local-first, sem banco remoto ou Supabase;
- preparar fontes publicas seguras sem crawler amplo nem login automatico.

## Limites de seguranca

- Nao foi criado login automatico.
- Nao foi criado bypass de CAPTCHA.
- Nao foi criado auto-apply.
- Nao foi alterada a logica sensivel de authenticated browser, Chromium ou CDP.
- O importador de URL tenta apenas leitura publica simples da URL informada.
- Se a pagina bloquear acesso, exigir login ou nao permitir leitura simples, a API orienta colar o
  texto da vaga manualmente.
- O usuario revisa o item antes de salvar em Candidaturas ou usar em analises.
- Dados e capturas ficam em armazenamento local ignorado pelo Git.

## Importadores

### Texto

`POST /sources/imports/text` recebe texto da vaga, URL opcional, empresa opcional, cargo opcional e
notas. O backend usa parser local, preserva o texto original e cria um item na Caixa de Entrada.

### Link

`POST /sources/imports/url` recebe uma URL e tenta leitura publica simples. Em bloqueio/login/erro,
cria item com status `error` e warning amigavel para colar texto manualmente.

### CSV

Campos esperados:

```csv
cargo,empresa,link,local,descricao,fonte,status,observacoes
Analista de Dados,Empresa Exemplo,https://example.com/jobs/123,Remoto,"Python, SQL e dashboards",CSV Manual,nova,"vaga ficticia"
Desenvolvedor Backend,Tech Exemplo,https://example.com/jobs/456,Hibrido,"APIs, testes e bancos de dados",CSV Manual,nova,"vaga ficticia"
```

Aliases aceitos:

```txt
cargo/title
empresa/company
link/url
local/location
descricao/descrição/description
fonte/source
observacoes/observações/notes
```

### JSON

Formato esperado:

```json
[
  {

    "observacoes": "vaga ficticia"

```

## Deduplicacao

A deduplicacao v1.7.0 e local e explicavel. Criterios:

- URL normalizada igual;
- empresa + cargo + localidade;
- empresa + cargo com similaridade simples;
- hash de texto normalizado.

Decisoes modeladas:

```txt
possible_duplicate
confirmed_duplicate
not_duplicate
```

Na UI v1.7.0, o usuario pode arquivar/ignorar/manter separado. Mescla profunda fica para versao
futura para evitar perda de historico.

## Integracao com tracker e Kanban

Ao salvar um item em Candidaturas:

- o backend cria uma entrada no tracker local;
- preserva link e origem;
- mapeia origem para collection method seguro;
- o Kanban mostra fonte/origem nos cards e na lista.

## IA nas importacoes

A v1.7.0 nao depende de IA para importar. Quando `use_ai=true` em texto manual, o backend registra
warning de que o enriquecimento por IA ainda e planejado e segue com extracao local. Assim:

- tudo funciona com IA desligada;
- nao ha segredo no frontend;
- nenhuma API key retorna ao navegador;
- requisitos nao sao inventados.
