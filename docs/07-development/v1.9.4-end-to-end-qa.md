# v1.9.4 - QA Ponta a Ponta

Checklist manual para validar a v1.9.4 em ambiente local.

## Preparação

1. Rodar `python scripts/run_api.py`.
2. Rodar `cd apps/web && npm run dev`.
3. Abrir o frontend em modo API Real.
4. Confirmar que a extensão v0.9.0 está carregada como extensão não empacotada, se o fluxo da extensão for testado.

## IA e providers

1. Abrir **Configurações / IA**.
2. Selecionar **Local seguro** e confirmar que IA externa fica desligada.
3. Selecionar **IA básica** e conferir fluxos principais habilitados.
4. Selecionar Gemini, escolher modelo conhecido, salvar e testar conexão.
5. Selecionar OpenAI, escolher modelo conhecido, salvar e testar conexão.
6. Usar modelo customizado apenas na seção avançada.
7. Confirmar que a UI não mostra API key após salvar.
8. Confirmar que `localStorage` e `sessionStorage` não guardam API key.

## Lattes e Perfil

1. Colar texto Lattes em **Perfil > Acadêmico / Lattes**.
2. Extrair com IA quando provider estiver configurado.
3. Confirmar que os itens são candidatos revisáveis.
4. Confirmar manualmente um item no Perfil Universal.

## Editais e Concursos

1. Colar edital mock em **Editais / Concursos**.
2. Marcar **Usar IA/Gemini se configurada** ou fluxo equivalente.
3. Confirmar `provider_used`, `requested_provider` e fallback quando aplicável.
4. Verificar órgão, banca, cargo, datas, taxa, requisitos e conteúdo programático.
5. Gerar Exam Fit Score usando Perfil Universal.
6. Gerar plano de estudo inicial.
7. Confirmar aviso de que o edital oficial sempre prevalece.
8. Confirmar ausência de botão de inscrição automática.

## Extensão e site

1. Abrir uma página pública de edital/concurso.
2. Clicar **Capturar edital / concurso** no popup.
3. Confirmar que a Local Companion recebe `kind=public_exam`.
4. Abrir **Fontes e Captura > Extensão Local**.
5. Ver a captura com tipo Edital/Concurso.
6. Clicar **Importar como Edital/Concurso**.
7. Confirmar abertura de `/public-exams?capture_id=...`.
8. Confirmar que o edital vira rascunho revisável, sem salvar automaticamente.
9. Clicar em **Status/contexto seguro** na extensão.
10. Confirmar que só há resumo seguro do Perfil, sem API key ou perfil inteiro.

## Segurança

1. Rodar grep/rg por chaves reais antes de commit.
2. Confirmar que `docs/ethics/allowed-vs-not-allowed.md` não existe.
3. Confirmar que segredos ficam em `.env*local`, `data/secrets/` ou variável de ambiente local.
4. Confirmar que nenhum teste externo roda sem env var explícita.
5. Confirmar que não houve alteração no fluxo authenticated-browser.
