# v1.9.4 - AI Providers, Model Catalog, Extension Bridge & QA

A v1.9.4 é uma release de confiabilidade e integração. Ela não cria um produto novo grande; fecha lacunas de configuração de IA, catálogo de modelos, OpenAI, uso real da IA em Editais/Concursos e ponte da extensão com o site.

## Entregas

- Configurações de IA com presets Local seguro, IA básica, IA completa e Personalizado.
- Catálogo de modelos Gemini/OpenAI com builtin, cache local e refresh opcional.
- OpenAI como provider real no backend local.
- Gemini/OpenAI usando o modelo selecionado nos runtimes estruturados.
- Editais/Concursos usando Prompt Registry quando `use_ai=true` e fallback local quando provider falha.
- Extensão com ação **Capturar edital / concurso** e payload `kind=public_exam`.
- Local Companion com `/capture/public-exam` e `/capture/context-summary`.
- Site importando captura como edital via `/public-exams?capture_id=...`.
- Contexto seguro do Perfil Universal para extensão, sem perfil completo, memória completa ou API key.
- Testes unitários/mockados e testes externos opt-in por variável de ambiente.

## Regras de segurança preservadas

- API key fica no backend local/env local.
- A extensão não lê chave do app.
- A extensão não captura cookies, tokens, sessão, headers ou storage de terceiros.
- Captura de edital vira rascunho revisável.
- Perfil Universal só recebe evidência confirmada manualmente.
- Editais não viram candidatura privada.
- Nenhum fluxo faz inscrição automática, pagamento ou envio de documento.
- O fluxo `/api/v1/sources/authenticated-browser/collect` não foi alterado.

## Testes de provider real

Use apenas ambiente local:

```bash
$env:SOTUHIRE_TEST_GEMINI_API_KEY="..."
pytest -m external_ai

$env:SOTUHIRE_TEST_OPENAI_API_KEY="..."
pytest -m external_ai
```

No Linux/macOS:

```bash
SOTUHIRE_TEST_GEMINI_API_KEY=... pytest -m external_ai
SOTUHIRE_TEST_OPENAI_API_KEY=... pytest -m external_ai
```

Sem essas variáveis, os testes são pulados.
