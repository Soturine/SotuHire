# v0.4 — Tracker, histórico e dashboard

## Histórico local

O usuário pode salvar uma análise após confirmar o aviso de privacidade. O tracker não salva o texto bruto do currículo.

## Status

```text
found
analyzed
good_fit
applied
message_sent
follow_up
interview
technical_test
rejected
offer
archived
```

O usuário pode abrir uma análise salva, rever scores, recomendação, data e alterar status.

## Dashboard

Exibe:

- total de vagas analisadas;
- média de Match Score;
- média de ATS Score;
- média de Opportunity Fit;
- vagas recomendadas para aplicar;
- vagas com alto risco;
- últimas análises.

## Persistência

O armazenamento padrão é JSON local:

```text
data/sotuhire-history.json
```

`data/` e `exports/` são ignorados pelo Git.

## Testes

- storage salva e lê;
- confirmação de privacidade é obrigatória;
- tracker altera status;
- status desconhecido é rejeitado;
- dashboard funciona vazio e calcula médias.

## Relação com a v0.4.1

A hotfix v0.4.1 preserva o tracker e o dashboard entregues na v0.4.0, mas reorganiza sua apresentação dentro da nova camada `modules/ui/`.

O histórico continua exigindo confirmação explícita antes de salvar. O texto bruto do currículo e da vaga continua fora dos registros persistidos.

As métricas do dashboard agora usam o mesmo tema, contraste e cards da revisão de currículo, revisão de vaga e resultado.

Consulte [v0.4.1 — UI/UX Hotfix + Parser Fixes](v0.4.1-ui-parser-hotfix.md) para os detalhes de interface, fluxo e parser.
